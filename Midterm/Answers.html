24: Explain how a using(){ } block functions and what types of objects it applies to

Answer: a using(){} block creates something that equates to an inline function. it comes with the intent that "the thing i'm doing right here isn't needed for this whole function, just for this specific thing". the merit in having a using(){} block is that it CLEANS UP many different things so that you don't have to worry about it. an example of where this is helpful is in querying a database. using takes care of things like closing up the connection. the types of objects that can be used in a using are those that inherit from IDisposable. this is an interface that requires you to implement a dispose() method for cleaning yourself up.


25: I often drone on about HTML being a semantic language. What do I mean, and how does this affect the way HTML should be written

Answer: HTML is a semantic language because it is meant to be human readable. the whole intent is that people can understand what the webpage is doing. an example of this, instead of just using <div> for everything, is to break things up into a <header>, <footer>, etc. this makes it so that the site can be interpreted to do the same thing for the machine, but when a person is reading it they are able to understand the purpose for each of the elements. this changes how HTML should be written because you should treat is as if you're writing an exam. be painfully clear and use all the tools you have to help do that.

61: With regards to middleware, explain what is meant by short-circuiting the pipeline and how it is achieved

Answer: short circuiting the pipeline is when a middleware prevents any middleware that occurs after it from being able to do anything. this is achieved by not calling await _next(context). this call is what moves you from the current middleware into the next one, so not calling it is what short-circuits the pipeline
